# Разработка

Внутренняя документация проекта. Для использования библиотеки смотри [README](../README.md).

## Команды

| команда | что |
|---|---|
| `npm run dev` | dev-сервер приложения (Vite) |
| `npm test` / `npm run test:watch` | vitest — 92 теста: детерминизм, roundtrip кодов, MIDI-инварианты, жанровые проверки, SMF |
| `npm run typecheck` | tsc --noEmit (strict) |
| `npm run build` | typecheck + сборка приложения |
| `npm run build:lib` | tsup → `dist/core.js`, `dist/audio.js` (ESM + d.ts) |
| `npm run demo` | headless-генерация .mid в Node (смоук headless-инварианта) |

## Структура

```
src/core/    детерминированное ядро (без DOM/Tone): PRNG, кодек серийника,
             теория, генераторы партий, жанры, SMF-экспорт
src/audio/   Tone.js: голоса-синтезаторы, мастер-FX по жанрам, live-плеер,
             offline-рендер, WAV/MP3-энкодеры
src/ui/      приложение: панель, piano-roll, embed-виджет
demo/        node-скрипт пакетной генерации
tests/       vitest
```

## Снапшот-канарейки

Фиксированный сид → отпечаток сгенерированного трека, по канарейке на жанр.
Изменился отпечаток = ранее выданные коды звучат иначе. До релиза это разрешено
(политика 2026-06-12): обновляй `vitest -u` + пометка в CHANGELOG. После релиза —
bump `CODE_VERSION` в `src/core/code.ts`.
