## 2026-06-12 — Keygen v2: разнообразие и канон cracktro

- **Мотив:** keygen — единственный флагманский жанр на чистых generic-генераторах; все сиды звучали «одинаково вверх», без главных маркеров трекерной сцены.
- **Сделано:**
  - Арп: пул форм цикла `up/updown/octaves/down/thumb` (`arp.patterns`, выбор per-имя-секции, `comping.ts`), акценты на старте цикла/доли.
  - Бас: два новых стиля в `bass.ts` — `octave8` (качель root/root+12 восьмыми, ГЛАВНЫЙ keygen-бас) и `syncopated16` (3-3-2 маска 16-ми, поздне-Amiga кач); per-seed выбор через `bass.styles` (3/2/1).
  - Драмы: `HALFTIME_BREAK` паттерн (обещан старым комментом, отсутствовал), трекерный snare-rush филл 32-ми (`drums.fillStyle: 'rush'`).
  - Гармония: 5 новых прогрессий — Andalusian и каденции с **мажорным V** (гармонико-минорная классика жанра), вамп i–VI, 8-аккордная с harmonic rhythm 2 бита, Vsus4→V; триадные quality `maj/min/sus2/sus4` в `chords.ts`; `distinctProgressions` — B гарантированно ≠ A (filter занятых + ОДИН weighted-draw, без re-roll циклов — ровно один шаг RNG).
  - Мелодия: кастомный hook `genKeygenLead` в `keygen.ts` — 1-барный мотив с повторами нот (da-da-da) и октавным попом, **транспонируется по аккордам** (а не random-walk), call/response парами (response разрешается в корень), скейл-раны 16-ми на стыках фраз, каденция секции.
  - Слайды: `NoteEvent.slide` (трекерный tone-portamento 3xx) — аудио-слой глайдит через `MonoSynth.setNote` без ретриггера огибающей; генератор обязан держать carrier-ноту звучащей весь слайд. MIDI-экспорт поле игнорирует.
  - Структуры: третий вариант с `break`-секцией (drums drop, играют arp+bass).
- **Решения / отвергнутые альтернативы (из обсуждения):**
  - 64-е арпы (rate 16, «честный 50Hz чип») — отвергнуто: keygen уже на 32-х (rate 8 = 18–24 нот/сек), 64-е через PolySynth дают кашу, не чип; настоящий C64-эффект 0xy — это трюк аудио-слоя (один осциллятор), не нотные события.
  - Модуляция финальной A (+2 полутона) — отвергнуто: ломает мемоизацию harmony/melody-кэша по имени секции и бесшовный луп.
  - Re-roll для B≠A — заменён на filter+pick по ревью пользователя (детерминированный расход RNG, гарантия уникальности).
- **Найденный баг:** `octave8` при старом регистре баса [33,50] вырождался в монотонный памп для половины корней (root+12 не влезал) → регистр keygen-баса расширен до [31,55].
- **Изоляция:** все новые RNG-вызовы строго за opt-in конфиг-полями. Проверено per-genre hash-сравнением до/после: 8 остальных жанров побайтово идентичны, изменился только keygen.
- **Снапшот-канарейка обновлена осознанно** (политика инварианта §2: до релиза коды ломать ОК, CODE_VERSION не bump'аем).
- **Verification:** tsc чисто, vitest 92/92; на 30 сидах: 30 уникальных песен, басы 15/12/3 по весам, 5 форм арпа, A≠B 30/30, внедиатоническая гармония (maj V/sus) в 20/30, halftime в 13/30, 155 слайдов — все покрыты carrier-нотой.
